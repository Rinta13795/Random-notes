```bash
# 2. 创建虚拟环境（名字通常叫 venv）
python3 -m venv venv

# 3. 激活虚拟环境

```

看任何项目的目录结构：

项目根目录/
├── schemas/      ← 数据模型（最重要，定义"是什么"）
├── services/     ← 业务逻辑（第二重要，定义"怎么做"）
├── api/          ← 接口层（对外暴露）
├── utils/        ← 工具函数（辅助功能）
└── tests/        ← 测试代码

看代码的顺序 = schemas → services → api
因为：先知道"数据是什么样的"，才能理解"怎么处理数据"


## `source ~/.zshrc`

**作用**: 重新加载你的shell配置文件

**详细解释**:

- `~/.zshrc` 是zsh的配置文件(你用的是zsh shell)
- 里面存了各种设置,包括**PATH环境变量**(告诉系统去哪找命令)
- 当你安装 `code` 命令时,VSCode会把路径写入这个文件
- 但**只有重启终端或运行 `source ~/.zshrc` 才会生效**



（- 空格，可以变成分点）
